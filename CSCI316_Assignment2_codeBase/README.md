## Project Overview

Sentiment analysis of Lebanese Arabizi and MSA Arabic tweets using
transfer learning with mBERT. Implements Full Fine-Tuning and LoRA (PEFT)
across PyTorch and TensorFlow/Keras frameworks.

**Task:** Sentiment Analysis in Code-Switched Text (Task 2)
**Model:** bert-base-multilingual-cased (mBERT)
**Labels:** Positive / Negative / Neutral

## Datasets

Both datasets are publicly available and must be downloaded before
running the notebooks.

### 1. mksaad Arabic Sentiment Twitter Corpus (Source Domain)

- **Download:** https://www.kaggle.com/datasets/mksaad/arabic-sentiment-twitter-corpus
- **Files needed:** `train_Arabic_tweets_positive_negative.tsv` and
  `test_Arabic_tweets_positive_negative.tsv`
- **Description:** 58K MSA Arabic tweets with binary pos/neg labels.
  Used for source domain pre-training before transfer to Arabizi.

### 2. Raïdy Arabizi Sentiment Dataset (Target Domain)

- **Download:** https://www.kaggle.com/datasets/mariajmraidy/datasets-for-sentiment-analysis-of-arabizi
- **File needed:** `3-class-sentiment-arabizi-ds.csv`
- **Description:** 1,799 Lebanese Arabizi tweets with 3-class labels
  (Positive/Negative/Neutral). This is the main target task dataset.

## Running the Notebooks

### Requirements

- Google Colab with T4 GPU (Runtime → Change runtime type → T4 GPU)
- Google Drive mounted
- Kaggle datasets downloaded and uploaded to Drive (see structure above)

### PyTorch Notebook (CSCI316_PyTorch.ipynb)

Covers preprocessing, source domain training, Full Fine-Tuning transfer,
LoRA/PEFT, evaluation, DTGS custom metric, and live demo.

**First time setup:**

1. Upload datasets to Drive following the folder structure above
2. Open notebook in Colab with T4 GPU
3. Run all cells in order — back-translation augmentation (~10 min)
   and source domain training (~40 min) only need to run once

**Resuming after disconnect:**

- Run the restore cell near the top which reloads all data from Drive
- Do NOT rerun the back-translation cell — load from saved CSV instead
- Do NOT rerun source domain training — load checkpoint from Drive instead

### TensorFlow Notebook (CSCI316_TensorFlow.ipynb)

Covers TF/Keras full fine-tuning, manual LoRA implementation,
DTGS evaluation, and framework comparison.

**Critical:** This notebook must be run in a completely separate
Colab session from the PyTorch notebook. TensorFlow and PyTorch
cannot be imported in the same session without causing a circular
import error in tf_keras.

**Setup:**

1. Open a NEW Colab notebook session (not the same as PyTorch)
2. The first cell sets `os.environ["TF_USE_LEGACY_KERAS"] = "1"`
   — this must run before any other import
3. Datasets are loaded from the same Drive paths as the PyTorch notebook

## Docker Deployment

### Prerequisites

- Docker installed: https://docs.docker.com/get-docker/
- Model checkpoint files (download from Drive or train using notebook)

### Download model files from Drive

Download these files from Google Drive and place them
in a local `models/` folder next to app.py:

```
models/
├── config.json
├── model.safetensors
├── tokenizer.json
├── tokenizer_config.json
├── vocab.txt
└── special_tokens_map.json
```

### Build and run

```bash
# Build the image
docker build -t arabizi-sentiment .

# Run the container
docker run -p 5000:5000 arabizi-sentiment
```

### Test the API

Health check:

```bash
curl http://localhost:5000/health
```

Predict sentiment:

```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"text": "hatha ktir 7elo ana mabsout ktir"}'
```

Expected response:

```json
{
  "input": "hatha ktir 7elo ana mabsout ktir",
  "cleaned": "hatha ktir حelo ana mabsout ktir",
  "sentiment": "Positive",
  "confidence": 0.7823,
  "scores": {
    "negative": 0.0891,
    "neutral": 0.1286,
    "positive": 0.7823
  }
}
```

## Key Results

| Strategy             | Framework | Macro F1 | Accuracy | Params       | Time |
| -------------------- | --------- | -------- | -------- | ------------ | ---- |
| No-Transfer Baseline | PyTorch   | 0.22\*   | 42%      | 178M         | 137s |
| Full Fine-Tuning     | PyTorch   | 0.63     | 68%      | 178M         | 138s |
| LoRA / PEFT ⭐       | PyTorch   | 0.65     | 68%      | 297K (0.17%) | 83s  |
| Full Fine-Tuning     | TF/Keras  | 0.66     | 69.4%    | 178M         | 205s |
| LoRA / PEFT          | TF/Keras  | 0.23†    | 50.1%    | ~2K          | 103s |

\*Baseline checkpoint loading issue in test session; trained val F1 was 0.62
†TF LoRA collapsed to majority-class prediction — result unreliable
⭐ Best efficiency-adjusted result

## Custom Metric: DTGS

Dialectal Transfer Gap Score measures F1 degradation as Arabizi
intensity increases. Lower = more robust to code-switching.

| Model                       | DTGS      |
| --------------------------- | --------- |
| No-Transfer Baseline        | 0.1023    |
| Full Fine-Tuning (PyTorch)  | 0.0668 ⭐ |
| LoRA (PyTorch)              | 0.0710    |
| Full Fine-Tuning (TF/Keras) | 0.1302    |
